hey, welcome to my weekly. hopefully you enjoy it :)

a few notes before you begin:

  • each puzzle page has a check answer box at the top of the page. answers will consist of only letters, and case does not matter (only letters are checked either way)

  • this check answer page has a small proof-of-work that will be completed automatically upon pressing the check answer button. this means that each answer will take a few seconds to be checked.

  • some puzzles have confirmations via the answer submission system, you can submit any answer you get and the website will tell you if you are making good progress

  • this weekly is partially quite technical. about half of the puzzles consist of a python script with the output of running the script, where you must try and recover the SECRET_MESSAGE in each case. you can use https://www.online-python.com to run them if you want. these puzzles are marked as technical, while ordinary puzzles are marked with puzzle.

  • note that any challenge that uses the Crypto library won't run on the site provided, in which case I have tried to make the functions simple enough that they can be understood without needing to run them.

  • to make this weekly more accessible, a few of the technical puzzles have slightly easier versions of them, which act as a stepping stone to solving the actual one. note that these puzzles are completely optional and their answers do not contribute to the metapuzzle.

  • no coding is strictly required to solve the technical puzzles, however it may help speed up the solving process.

  • no files are hidden (i.e. you do not need to change the url to access a part of the puzzle), and nothing will be hidden in the source code either.

  • there is a metapuzzle at the bottom of the puzzles list. this puzzle is designed to be solved last, and requires the answers of all the other puzzles (although you donâ€™t necessarily need all of the answers to solve it!)
